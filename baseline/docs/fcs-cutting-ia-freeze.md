# 工艺工厂运营系统 - 裁片管理菜单重构方案（冻结版）

## 1. 文档定位

本文档是“工艺工厂运营系统 - 裁片管理菜单重构与页面职责边界”的冻结基线。  
后续 PRD、原型、Codex Prompt、页面开发、路由调整，均以本文档为准。  
若会议纪要、旧页面、旧文案与本文冲突，以本文为准。

本文档只负责冻结信息架构、canonical 命名、路由口径与页面职责边界，不展开后续阶段的业务细节实现。

## 2. 适用范围

本文档只覆盖：

- 工艺工厂运营系统中的裁片域
- 裁片域菜单、页面命名、canonical 路由与 alias 兼容规则
- 裁片域 14 个页面的对象边界与职责划分

本文档不直接改动以下系统的职责边界，但会说明与它们的协同关系：

- 平台运营系统
- 工厂端（移动应用 / PDA）

## 3. 三端职责边界

### 3.1 平台运营系统

平台运营系统承接跨厂监控、汇总、审计、异常、结算与中台视角。  
平台关注的是跨工厂、跨任务、跨结算周期的统一视图，而不是裁片专厂现场执行细节。

### 3.2 工艺工厂运营系统

工艺工厂运营系统承接裁片专厂运营视角，包括：

- 裁片排产与执行准备
- 原始裁片单管理
- 仓库配料与领料衔接
- 裁片仓交接与周转载具流转管理
- 补料、特殊工艺与裁剪总结收口

本系统关注“裁片专厂如何组织对象、推进执行、完成仓交接和异常收口”，不承担跨厂平台审计角色，也不承担 PDA 现场扫码执行角色。

### 3.3 工厂端（移动应用 / PDA）

工厂端（移动应用 / PDA）承接现场扫码、现场执行、现场交接和现场录入。  
PDA 关注的是动作发生、扫码确认、现场状态回写，不承担裁片域对象建模和中台级对象归属判断。

## 4. 裁片域冻结规则

### 4.1 规则 1~6

1. 生产单来源唯一：生产需求单。一个生产需求单对应一个生产单。
2. 生产单不得因仓库配料、领料、裁剪安排而生成、拆分或合并。
3. 裁床现场“可裁判断”发生在原始裁片单层，不发生在生产单层。
4. 合并只允许发生在裁片执行层；合并对象是多个原始裁片单，合并产物是“合并裁剪批次”。
5. 同一生产单下的不同原始裁片单，可分批进入不同合并裁剪批次。
6. 菲票永远回落原始裁片单；合并裁剪批次只作为执行上下文，不作为菲票归属主体。

### 4.2 规则 7~10

7. 中转袋/周转箱是独立载具实体，每个载具有唯一编码，可循环复用。
8. 中转袋在单次使用周期内，通过“扫描载具码 + 扫描裁片菲票”建立与裁片明细的父子映射关系。
9. 一个车缝任务可对应一个或多个中转袋；单个中转袋在单次使用周期内只归属一个车缝任务。
10. 车缝完成后，成衣需通过中转袋回货入仓；系统必须记录中转袋的发出、签收、回仓与复用周期，确保载具与实物流转可追溯。

## 8. 页面职责边界

### 8.1 生产单进度

本页负责：

- 以生产单为主对象查看裁片域总体进度
- 汇总查看该生产单关联的配料、领料、裁剪与异常概览
- 为下游页面提供总览入口

本页不负责：

- 判断是否可裁
- 直接组织原始裁片单合批
- 承载菲票归属

### 8.2 可裁排产

本页负责：

- 在原始裁片单层进行可裁判断
- 按同款与原始裁片单组织裁床排产选择

本页不负责：

- 在生产单层做可裁判断
- 合并生产单
- 承接菲票归属

### 8.3 合并裁剪批次

本页负责：

- 管理多个原始裁片单在执行层的合并裁剪批次
- 记录合批对象与执行上下文

本页不负责：

- 改变原始裁片单身份
- 改变生产单归属
- 承接菲票最终归属

### 8.4 裁片单（原始单）

本页负责：

- 管理原始裁片单身份
- 记录原始裁片单与生产单、面料、唛架、执行记录的关联
- 承接原始裁片单维度的查看与追溯

本页不负责：

- 把原始裁片单直接等同于生产单
- 把合并裁剪批次当作主对象
- 替代合批执行页

### 8.5 仓库配料 / 领料

本页负责：

- 承接配料、领料与相关单据状态
- 连接仓库与裁床之间的物料衔接
- 查看与原始裁片单相关的配料/领料状态

本页不负责：

- 生成生产单
- 决定可裁判断
- 代替仓交接页面

### 8.6 唛架 / 铺布

本页负责：

- 管理唛架信息
- 管理铺布模式、铺布记录及其与后续补料判断相关的数据

本页不负责：

- 执行层合批
- 菲票打印归属管理
- 仓储交接

### 8.7 菲票 / 打编号

本页负责：

- 承接菲票打印与打编号入口
- 管理二维码基础链路
- 保证菲票打印归属回落原始裁片单

本页不负责：

- 把菲票归属到合并裁剪批次
- 作为仓交接主页面
- 承接中转袋映射主档

### 8.8 裁床仓

本页负责：

- 查看裁床侧配置面料库存与剩余面料库存
- 管理与裁床仓相关的入出仓和签收视图

本页不负责：

- 代替裁片仓
- 代替样衣仓
- 代替中转袋交接主档

### 8.9 裁片仓

本页负责：

- 管理裁片完成后的入仓、分区、查找与后道交接准备
- 承接裁片仓对象的库存与流转状态

本页不负责：

- 管理裁床仓库存
- 管理样衣仓
- 管理中转袋复用周期

### 8.10 样衣仓

本页负责：

- 管理样衣存放、调用、归还与流转记录

本页不负责：

- 承接裁片仓库存
- 代替中转袋交接
- 承接菲票打印链路

### 8.11 中转袋 / 车缝交接

本页负责：

- 管理中转袋 / 周转箱主档
- 管理口袋单次使用周期
- 管理扫描载具码 + 扫描裁片菲票建立的父子映射
- 管理车缝任务绑定、发出、签收、回仓与复用追溯

本页不负责：

- 代替裁片仓入仓页面
- 承接生产单主进度
- 替代 PDA 现场扫码执行

### 8.12 补料管理

本页负责：

- 基于铺布与裁剪差异管理补料建议、审核与回写动作

本页不负责：

- 代替铺布记录页
- 代替生产单总览
- 代替特殊工艺管理

### 8.13 特殊工艺

本页负责：

- 管理需单独承接的裁片厂特殊工艺任务与附加记录

本页不负责：

- 承接全部异常收口
- 代替补料管理
- 代替平台侧异常中心

### 8.14 裁剪总结

本页负责：

- 汇总生产单、原始裁片单、合并裁剪批次、仓储与异常
- 形成裁片域收口视图

本页不负责：

- 代替单对象执行页
- 代替可裁排产
- 代替仓交接主页面

## 9. 命名冻结规则

- 订单进度
- 裁片单（未注明原始单）
- 仓库管理
- fei-ticket
- fei-list
- tasks
- orders
- stats
- bed-stats
- cutting-summary

这些名称仅允许作为兼容入口或历史文件名存在，不得反向污染 canonical 页面命名。

## 11. 使用说明（给 PRD / 原型 / Codex / 开发）

1. 新增页面、调整路由、修改 breadcrumb 前，先对照本文档确认页面主对象和责任边界。
2. 若旧路由仍需保留，必须按“alias 仅兼容、canonical 才是主入口”处理。
3. 若某个页面开始承接新的对象，请先判断是否已越出本文档定义的页面职责。
4. 若 PRD、原型或 prompt 中出现旧名称，应先映射到本文档中的 canonical 名称，再继续实现。
5. 后续阶段建设应优先基于本文档中 14 个页面的职责边界推进，不再退回“单一裁片管理 + 平铺页面”的旧信息架构。
